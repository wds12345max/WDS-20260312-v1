# OpenClaw 瑜伽 App 开发可用能力清单 & 边界说明
## 前言
本文档明确 OpenClaw 在瑜伽 App 开发项目中的可用能力范围、边界限制和注意事项，用于规范协作流程，提高开发效率，避免沟通误解。

适用范围：瑜伽 App 全生命周期开发（需求→设计→开发→测试→部署→运维）

## ⚠️ 能力边界说明
### 明确不可做的事项
1. **硬件相关操作**
   - 无法进行真机调试、设备连接测试
   - 无法操作物理硬件设备
   - 无法进行实际的线下设备测试

2. **敏感信息操作**
   - 无法处理支付配置、密钥填写、证书上传等敏感操作
   - 无法登录第三方平台账号（阿里云、微信开放平台等）
   - 无法获取和使用你的个人/企业敏感信息

3. **需要人工审核的操作**
   - 无法完成应用商店上架审核、资质提交等需要人工操作的流程
   - 无法进行合同签署、法律相关的操作
   - 无法代替你做业务决策

4. **超出技术范围的操作**
   - 无法进行 UI 设计图的视觉设计（只能基于设计图实现代码）
   - 无法进行音视频内容的创作和剪辑
   - 无法提供法律咨询、财务咨询等非技术类专业服务

### 能力限制说明
1. **代码质量**：生成的代码为示例代码，需要开发人员进行二次开发和测试，不能直接用于生产环境
2. **方案准确性**：技术方案为建议方案，需要结合实际业务场景进行评估和调整
3. **信息时效性**：涉及第三方服务的 API、文档等可能存在时效性问题，使用前需要核实最新版本

## 📝 注意事项
### 协作规范
1. 需求沟通尽量清晰、具体，提供尽可能多的背景信息
2. 复杂需求尽量拆分多个小任务，分阶段实现
3. 代码生成后需要人工进行 review 和测试，确保符合业务需求
4. 涉及敏感信息的操作，请你手动执行，我会提供完整的操作指南

### 安全提示
1. 不要向我提供任何敏感信息（密钥、密码、证书等）
2. 重要操作执行前会进行预览确认，确认无误后再执行
3. 所有代码提交都会有清晰的提交说明，方便追溯和回滚

### 沟通流程
1. 提出需求 → 我返回执行计划预览 → 你确认 → 我执行 → 反馈结果
2. 执行过程中遇到问题会及时同步，调整方案
3. 完成后会进行结果验证，确保符合预期
